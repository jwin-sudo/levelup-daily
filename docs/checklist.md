# Build Checklist

## Build Preferences

- **Build mode:** Step-by-step
- **Comprehension checks:** Yes — after each item, the agent asks a quick gut-check question
- **Git:** Commit after each item with message: "Complete step N: [title]"
- **Verification:** Yes — after each item, run the app and confirm what you see before moving on
- **Check-in cadence:** Learning-driven — talk through what was built and why before moving to the next step

- [x] **3. Journal router — conversational loop**
  Spec ref: `spec.md > Backend > Routers > routers/journal.py` (GET /journal/today, POST /journal/message)
  What to build: Create `routers/` directory with `__init__.py`. Write `routers/journal.py`. Implement `GET /journal/today`: looks up today's `JournalEntry` by date. If none: calls `claude.get_opening_message(goals, stats)`, creates entry with first AI message, returns `{status: "active", messages: [...]}`. Implement `POST /journal/message`: appends user message, calls `claude.send_message(messages, goals, stats, recent_debriefs)`, appends AI reply, saves entry, returns `MessageResponse`. On Claude failure: HTTP 503 with `{"detail": "AI unavailable — please try again."}`. Register the journal router in `main.py`.
  Acceptance: `GET /journal/today` returns an active session with the AI's opening message. `POST /journal/message` with `{"content": "I had a great day"}` returns an AI reply. Hitting `GET /journal/today` again returns the same session with both messages in history (not a new session).
  Verify: Use Swagger UI at `http://localhost:8000/docs`. Hit `GET /journal/today` — confirm response has `status: "active"` and one AI message. - [x] **4. Journal router — analysis and atomic completion**
  Spec ref: `spec.md > Backend > Routers > routers/journal.py` (POST /journal/complete)
  What to build: Implement `POST /journal/complete` in `routers/journal.py`. No body. Marks today's entry `completed=True`. Calls `claude.analyze_entry(messages, goals, stats, recent_debriefs)` → receives `DebriefAnalysis` dict. Applies all updates atomically: `JournalEntry.xp_awarded = analysis["xp_awarded"]`, `UserStats.xp_total += analysis["xp_awarded"]`, `UserStats.last_journaled_date = today` then calls `streak.update_streak(stats)`, applies each trait delta to the corresponding `TraitScore` row, creates new `PathMilestone` rows from `analysis["path_milestones"]`. Returns full `DebriefResponse` including `weekly_streak` (7-item boolean list, Mon–Sun) and `streak_count`. On Claude failure: HTTP 503, entry stays completed but debrief shows generic error — do not leave entry in broken state.
  Acceptance: `POST /journal/complete` returns a `DebriefResponse` with `xp_awarded` (5–25), `strengths`, `areas_to_improve`, `suggestions`, `trait_deltas` for all five traits, `streak_count` of 1, and `weekly_streak` with today's day set to true. Querying `UserStats` in the DB shows `xp_total` incremented and `last_journaled_date` set to today. `TraitScore` rows show updated scores.
  Verify: Run the full test sequence in order — 1) `GET /journal/today` (creates entry + opening message), 2) `POST /journal/message` with `{"content": "I had a really productive day working on a tough problem"}`, 3) `POST /journal/complete`. Confirm the DebriefResponse has all fields populated. Then check the DB: `SELECT xp_total, streak_count, last_journaled_date FROM userstats;` and `SELECT trait, score FROM traitscore;` — confirm XP was added and at least some trait scores incremented.
